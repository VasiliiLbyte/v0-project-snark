# Фаза 2 — Definition of Done

## Маршруты и layout

- [ ] Маршруты `dashboard`, `contacts`, `documents`, `profile` открываются как файловые маршруты App Router.
- [ ] В защищённой зоне используется только один layout: `app/(main)/layout.tsx`.
- [ ] `app/page.tsx` не содержит state-based роутинг и не рендерит дубли навигации.

## Авторизация и роли

- [ ] Middleware применяет проверку сессии ко всем защищённым маршрутам.
- [ ] Маршрут `/admin` доступен только ролям `admin` и `hr_manager`.
- [ ] Навигация может скрывать пункты по роли, но security-граница остаётся в middleware.

## Навигация App Router

- [ ] Sidebar использует `Link` для переходов между маршрутами.
- [ ] Активный пункт меню определяется через `pathname`.
- [ ] Обновление страницы не ломает активное состояние пункта меню.

## Server/Client границы

- [ ] Route-level `page.tsx` используют server-first загрузку данных.
- [ ] Интерактивные компоненты (поиск, фильтры, tabs, модалки) остались клиентскими.
- [ ] Нет hydration mismatch после разделения server/client компонентов.

## Контракт данных и будущие API

- [ ] Используется слой `repository` для доступа к данным вместо inline массивов в UI.
- [ ] Для доменов подготовлены hooks с единым контрактом: `data`, `isLoading`, `error`, `refetch`.
- [ ] Подготовлены типы и mapper-слой для замены mock-источника на API без переписывания UI.

## Тесты и валидация

- [ ] Smoke-сценарии покрывают базовый рендер основных разделов.
- [ ] Сценарий `login -> protected route -> dashboard` проходит стабильно.
- [ ] Прямой вход на защищённые страницы без сессии ведёт на `/login`.
